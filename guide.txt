J'ai examiné l'erreur et il semble qu'il s'agisse d'un problème de syntaxe dans le modèle d'affichage des erreurs intégré de Laravel, plus précisément dans vendor/laravel/framework/src/Illuminate/Foundation/resources/exceptions/renderer/components/context.blade.php. Apparemment, un espace est manquant avant un attribut de classe dans une balise <span> qui utilise data-tippy-content. Cela provoque une erreur secondaire lorsque Laravel essaie d'afficher une erreur primaire (comme celle qui pourrait se produire lors de la sauvegarde d'un article).

Comme je ne peux pas modifier les fichiers dans le répertoire vendor, je ne peux pas corriger cela directement.

Voici ce que je vous recommande de faire :

Vérifiez manuellement le fichier Vendor : Ouvrez le fichier /var/www/vendor/laravel/framework/src/Illuminate/Foundation/resources/exceptions/renderer/components/context.blade.php dans votre projet. Recherchez une ligne similaire à <span data-tippy-content="{{ $key }}"class="..."> (autour de la ligne 26, d'après l'erreur de la vue compilée). Si vous la trouvez, ajoutez un espace : <span data-tippy-content="{{ $key }}" class="...">.
Réinstallez les dépendances : Si vous n'êtes pas sûr de modifier ou si le fichier semble correct, le répertoire vendor pourrait avoir été corrompu. Essayez de supprimer le dossier vendor et d'exécuter composer install depuis la racine de votre projet pour obtenir de nouvelles copies des fichiers du framework.
Effacez à nouveau les caches : Après avoir essayé l'une de ces étapes, effacez tous les caches pertinents : php artisan view:clear, php artisan cache:clear, php artisan config:clear.
Une fois ces étapes suivies, essayez à nouveau de sauvegarder un article. Si la page d'erreur de syntaxe a disparu, nous saurons s'il y avait un autre problème sous-jacent avec la sauvegarde de l'article elle-même.
